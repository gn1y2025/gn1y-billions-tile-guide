Step 1

Windows Agent Doctor

Agent Doctor does not claim Tiles, does not install anything, does not update anything, and does not touch private keys. It only checks your setup and prints what to do next.

Copy-paste command — recommended Recommended for all users

This downloads the Doctor script, opens it in Notepad for review, then runs it.

PowerShell 
$DoctorUrl = "https://guide-by-gn1y.vercel.app/scripts/windows-agent-doctor.ps1"
$DoctorFile = Join-Path $env:TEMP "gn1y-windows-agent-doctor.ps1"

Invoke-WebRequest -UseBasicParsing -Uri $DoctorUrl -OutFile $DoctorFile

Write-Host ""
Write-Host "Windows Agent Doctor downloaded to:"
Write-Host $DoctorFile
Write-Host ""

notepad $DoctorFile

Read-Host "Review the script in Notepad. Press Enter to run Windows Agent Doctor"

powershell -NoProfile -ExecutionPolicy Bypass -File $DoctorFile

Find and check identity skill Use when your agent folder exists
PowerShell 
$AgentRoot = Read-Host "Paste the FULL path to your OpenClaw agent folder"

if (!(Test-Path $AgentRoot)) {
  throw "STOP: Agent folder does not exist: $AgentRoot"
}

$SkillDir = Get-ChildItem -Path $AgentRoot -Directory -Recurse -Filter "verified-agent-identity" |
  Select-Object -First 1 -ExpandProperty FullName

if (!$SkillDir) {
  throw "STOP: verified-agent-identity skill was not found. Open guides/existing-agent-status.md"
}

Set-Location $SkillDir

npm install

node scripts\getIdentities.js

Safety

Security rules for beginners

The guide is designed to avoid accidental paid claims and private data leaks.

Never share

  • seed phrase
  • private key
  • kms.json
  • API keys
  • full unredacted terminal logs

Redact before posting logs

  • local Windows username
  • local folder paths
  • DID / publicKeyHex if you want privacy
  • wallet addresses if you want privacy
  • tx hashes if you want privacy
